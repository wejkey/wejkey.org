Valentine - 1.4-dev1

Developed: 1.20.2
Tested: 1.16, 1.17, 1.18.2, 1.19, 1.20.2, 1.21

All those hearts in that one day. Hugs & kisses, marriages etc..

Features:

  • Changes custom words with custom symbol and color (in chat)
  • You can send a Kiss/Hug to a player (with cooldown)
  • Heart particles around player
  • Hugs/Kisses leaderboard
  • Multilanguage support
  • Marriage system
  • Permissions

Commands:

Command Alias Description Permission
/valentines /val, /vday Displays plugin's commands valentine.use
/valentinesplayer (player) /vstats (player) Displays player's stats valentine.use
/valentinesleaderboard /vlb Displays leaderboard valentine.use
/hug (player) /valstats, /vstats Hugs player valentine.use
/kiss (player) /valstats, /vstats Kisses player valentine.use
/marry (player) - Marry another player valentine.marry
/divorce - Divorce your partner valentine.divorce
/couples - Display all married couples valentines.use
/valtest - Plugin testing command CONSOLE

Permissions:

Permission Description
valentines.use Allows the player to use basic Valentines commands
valentines.effect Allows the player to see particle effects around them
valentines.marry Allows the player to marry other players
valentines.divorce Allows the player to divorce their partner
valentines.admin Allows access to administrative commands
CONSOLE Command works only in console

config.yml:

Copied!
          lang: "EN"

          # Messages
          Prefix: "&c❤&f&lValentines&c❤&r "
          NoPermissionMessage: "&cYou do not have permission to use this command."
          
          Hug: "&fYou hugged &d{player} ❤"
          HugReceived: "&d{player} &fhugged you&d! ❤"
          Kiss: "&fYou kissed &d{player} ❤"
          KissReceived: "&d{player} &fkissed you&d! ❤"
          
          Marriage:
            proposal-sent: "&fMarriage proposal sent to &d{player}"
            proposal-received: "&d{player} &fwants to marry you. Type &d/marry confirm &ffor confirmation."
          
            already-married: "&cYou are already married!"
            target-already-married: "&c{player} is already married!"
          
            no-pending-proposals: "&cYou have no pending marriage proposals!"
            proposer-offline: "&cThe player who proposed is no longer online!"
          
            accept-sender: "&d{player} &faccepted your marriage. You're together&d! ♥"
            accept-receiver: "You married &d{player} &fYou're together&d! ♥"
            decline-sender: "&c{player} doesn't want to be with you :("
            decline-receiver: "&cYou don't want to be with {player}"
          
            divorce-initiator: "&cYou have divorced {player}!"
            divorce-target: "&c{player} has divorced you!"
            divorce-confirm: "&fAre you sure you want to break up with &d{player}&f? Type &d/divorce &fagain to confirm."
          
            not-married: "&cYou are not married!"
            not-married-to: "&cYou are not married to {player}!"
          
            # Marriage prefix settings
            prefix:
              enabled: true
              position: "after"  # symbol "after" or "before" player's name
              symbol: "❤"
              color: "&c"
              tooltip: "&dMarried to &f{partner}"
              spacing: " "  # Space between name and symbol
          
          reload-success: "&aConfiguration and languages reloaded successfully!"
          reload-error: "&cError reloading configuration: {error}"
          
          cooldown-message: "&cYou can only {command} once every &d{time}&c seconds!"
          
          # Chat Replacement
          symbol-change: true
          symbol-color: "&c"
          symbol: "♥"
          words:
            - "love"
            - "valentines"
            - "heart"
          
          # Effects
          effect:
            enabled: true
            particle-density: 1
            update-rate: 2
            max-height: 2.0
            start-height: 0.1
            radius: 0.85
          
          enabled-worlds:
            - world
            - world_nether
          
          # Cooldowns (seconds)
          cooldowns:
            hug: 60
            kiss: 60

DisenchantedBook - 1.0-exp

Developed: 1.20.2
Tested: 1.20.2

You can disenchant your enchanted item by holding it and using command. You will be given a book of your item's disenchanted enchants.

Features:

  • Disenchant your enchanted item and get a book with the enchants

Commands:

Command Alias Description Permission
/disenchant - Disenchants item disenchantedbook.use

Permissions:

Permission Description
disenchantedbook.use Allows to use all commands

config.yml:

Copied!
          messages:
          prefix: "&a•&fDisenchantedBook&a•&r "
          no-permission: "&cYou don't have permission to use this command!"
          no-item: "&cYou must hold an item in your main hand!"
          no-enchantments: "&cThis item doesn't have any enchantments!"
          success: "&aSuccessfully disenchanted your item! Check your inventory for the enchanted book."
          inventory-full: "&cYou don't have enough inventory space! Disenchanting cancelled."
